# 5-bit S-box DFA Repository

🚀 **Fault Injection Experiments on SHAMASH, Ascon, ISAP, and Sycon**

This repository provides the C++ source code and experiment output files for conducting **Differential Fault Analysis (DFA)** on lightweight authenticated encryption schemes with **5-bit S-boxes**. The focus of this work is on fault injection and the statistical recovery of S-box inputs under bit-stuck fault models. This repository specifically targets algorithms like **SHAMASH**, **Ascon**, **ISAP**, and **Sycon**.

------

## 📁 Repository Contents

| File             | Description                                                  |
| :--------------- | :----------------------------------------------------------- |
| `shamashcpp.cpp` | Implementation of fault injection specifically targeting the SHAMASH cipher. |
| `main.cpp`       | DFA experiments for Ascon, ISAP, and Sycon using nibble-based fault models. |
| `.xlsx output`   | Excel results generated by each experiment, utilizing libxl for file manipulation. |

------

## 🛠️ Requirements

- **C++17 compiler**: Ensure you're using a compiler that supports C++17 standards.
- **libxl**: This library is used for reading/writing `.xlsx` files.
  - Make sure to link the library correctly in your project.
  - If you're using the trial version, ensure you set your license key properly to avoid any restrictions.

------

## ▶️ Usage Instructions

### 1. Configure libxl

Set your license key (if using the trial version) and adjust paths accordingly in the code.

### 2. Build and Run

Compile and run `main.cpp` and/or `shamashcpp.cpp` to conduct the DFA experiments.
These experiments simulate fault injections and statistical recovery of S-box inputs.

### 3. Output

The results of the experiments will be saved in an **Excel (.xlsx)** format, containing the following:

- **S-box values**: Both original 5-bit S-box inputs and faulted 4-bit inputs.
- **Fault values**: The specific faults injected during the experiments.
- **Round count and nibble stats**: Details about the rounds and nibble-level fault statistics.
- **Per-trial intersection logs**: Logs showing how many fault injections are needed to uniquely identify all S-box inputs.

------

## 📊 Example Output

The generated `.xlsx` file includes detailed information on each experiment:

- **Raw S-box input values**: Includes both original 5-bit inputs and the faulted 4-bit values.
- **Fault injection patterns**: Describes how faults were applied to each round of the cipher.
- **Differential output**: Shows the differences observed in the output due to fault injections.
- **Input recovery statistics**: Includes per-nibble recovery stats, as well as averages over the 64 S-boxes.
